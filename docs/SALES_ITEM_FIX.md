# إصلاح مشكلة إضافة البنود في صفحة المبيعات

## المشكلة 🔴

عند محاولة إضافة بند جديد في صفحة "العمليات الجراحية والمبيعات"، كان الزر معطلاً أو لا يعمل بشكل صحيح، خاصة عندما:
- نوع العملية هو "جراحة" أو "بيع"
- المستخدم لديه صلاحية عرض الأسعار
- حقل "سعر البيع" فارغ أو غير محدد

## السبب الجذري 🔍

كانت هناك عدة مشاكل:

1. **عدم التحقق من سعر البيع في شرط تعطيل الزر**: كان الزر يُعطّل فقط إذا كان `itemId` أو `quantity` فارغين، لكن لم يكن هناك تحقق من `sellingPrice` في حالة العمليات الجراحية.

2. **القيمة الافتراضية لسعر البيع**: عند اختيار صنف، إذا لم يكن له سعر بيع محدد، كانت القيمة تُعيّن إلى سلسلة فارغة `''` بدلاً من `'0'`.

3. **عدم التحقق في دالة handleAddItem**: لم تكن الدالة تتحقق من وجود سعر البيع قبل إضافة البند.

## الحل ✅

تم إجراء ثلاثة تعديلات رئيسية:

### 1. تحديث شرط تعطيل زر "إضافة للبند"

```tsx
// قبل الإصلاح
disabled={!currentItem.itemId || !currentItem.quantity}

// بعد الإصلاح
disabled={
  !currentItem.itemId || 
  !currentItem.quantity || 
  (formData.type !== 'usage' && canViewPrices && !currentItem.sellingPrice)
}
```

الآن الزر يُعطّل إذا:
- لم يتم اختيار صنف
- لم يتم إدخال كمية
- **أو** إذا كان نوع العملية ليس "استهلاك داخلي" والمستخدم لديه صلاحية عرض الأسعار ولم يتم إدخال سعر البيع

### 2. تحسين معالجة سعر البيع عند اختيار الصنف

```tsx
// قبل الإصلاح
sellingPrice: selectedItem.sellingPrice?.toString() || '',

// بعد الإصلاح
sellingPrice: (selectedItem.sellingPrice || 0).toString(),
```

الآن عند اختيار صنف، إذا لم يكن له سعر بيع محدد، يتم تعيين القيمة إلى `'0'` بدلاً من سلسلة فارغة.

### 3. إضافة التحقق في دالة handleAddItem

```tsx
// إضافة التحقق الجديد
if (formData.type !== 'usage' && canViewPrices && !currentItem.sellingPrice) {
  toast.error('يجب إدخال سعر البيع');
  return;
}
```

الآن إذا حاول المستخدم إضافة بند بدون سعر بيع في عملية جراحية أو بيع، سيظهر له رسالة خطأ واضحة.

## النتيجة 🎉

- ✅ الزر "إضافة للبند" يعمل بشكل صحيح
- ✅ لا يمكن إضافة بنود بدون سعر بيع في العمليات الجراحية والمبيعات
- ✅ يمكن إضافة بنود بدون سعر بيع في حالة "الاستهلاك الداخلي"
- ✅ رسائل خطأ واضحة للمستخدم
- ✅ تجربة مستخدم محسّنة

## الاختبار 🧪

للتحقق من أن الإصلاح يعمل:

1. افتح صفحة "العمليات الجراحية والمبيعات"
2. اضغط على "تسجيل عملية"
3. اختر نوع العملية "جراحة"
4. اختر صنفًا من القائمة
5. أدخل الكمية
6. تأكد من أن حقل "سعر البيع" يحتوي على قيمة
7. اضغط على "إضافة للبند"
8. يجب أن يُضاف البند بنجاح

## ملاحظات إضافية 📌

- في حالة "الاستهلاك الداخلي"، لا يظهر حقل "سعر البيع" ولا يُطلب
- في حالة "الجراحة" أو "البيع"، حقل "سعر البيع" إلزامي إذا كان للمستخدم صلاحية عرض الأسعار
- القيمة الافتراضية لسعر البيع عند اختيار صنف هي سعر البيع المحدد في المخزون، أو `0` إذا لم يكن محددًا
