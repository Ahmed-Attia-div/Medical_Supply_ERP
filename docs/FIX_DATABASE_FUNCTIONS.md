# 🔧 إصلاح خطأ الفاتورة - تشغيل Database Functions

## المشكلة
الخطأ: `Could not find the function create_invoice_transaction`

هذا يعني أن الدالة غير موجودة في قاعدة البيانات Supabase.

## ✅ الحل (خطوات بسيطة)

### الطريقة 1: من خلال Supabase Dashboard (الأسهل)

1. **افتح Supabase Dashboard**
   - اذهب إلى: https://supabase.com/dashboard
   - اختر مشروعك

2. **افتح SQL Editor**
   - من القائمة الجانبية، اضغط على "SQL Editor"
   - أو اذهب مباشرة إلى: https://supabase.com/dashboard/project/YOUR_PROJECT_ID/sql

3. **انسخ والصق الكود**
   - افتح الملف: `update_invoice_payment_edit.sql`
   - انسخ **كل** محتويات الملف
   - الصقه في SQL Editor

4. **شغّل الكود**
   - اضغط على زر "Run" أو Ctrl+Enter
   - انتظر رسالة "Success"

5. **جرب الفاتورة مرة أخرى**
   - ارجع للصفحة وحاول إنشاء فاتورة جديدة
   - المفروض تشتغل بدون مشاكل! ✅

---

### الطريقة 2: باستخدام Supabase CLI (للمطورين)

إذا كان عندك Supabase CLI مثبت:

```bash
# تأكد من تسجيل الدخول
supabase login

# شغّل الملف
supabase db push --file update_invoice_payment_edit.sql
```

---

## 📋 ما الذي سيتم إنشاؤه؟

عند تشغيل الملف، سيتم:

1. ✅ إضافة عمود `payment_type` لجدول `invoices` (إذا لم يكن موجود)
2. ✅ إنشاء دالة `create_invoice_transaction` (لحفظ الفواتير)
3. ✅ إنشاء دالة `update_invoice_item_details` (لتعديل البيانات)

---

## 🎯 التأكد من النجاح

بعد تشغيل الملف، جرب:

1. إنشاء فاتورة جديدة
2. اختيار حالة الدفع (نقدي أو آجل)
3. إضافة أصناف
4. حفظ الفاتورة

إذا نجحت العملية، معناها كل شيء تمام! 🎉

## 📝 ملاحظة مهمة

هذا الملف آمن تماماً للتشغيل لأنه:
- يستخدم `CREATE OR REPLACE` (لن يحذف بيانات موجودة)
- يستخدم `IF NOT EXISTS` للعمود (لن يسبب خطأ إذا كان موجود)
- مجرد إنشاء دوال مساعدة للنظام

- ✅ طباعة الفواتير
- ✅ تعديل بيانات الفواتير
